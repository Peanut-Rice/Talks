SCENE 01 · 已有能力

制度合规逐条比对

选定被审计单位后，AI 自动检索该单位应执行的制度清单，逐条比对内部制度与外部法规，判定每条义务的覆盖状态。

执行流程

① 信息提取 Skill 检索制度清单（集团制度 + 行业法规 + 子公司细则），剔除废止条款
② 合规对齐 Skill 逐条提取义务条款，结构匹配内外条款
③ 输出覆盖矩阵（✅/⚠️/❌），未覆盖项生成审计疑点
④ 检索同类案例，生成整改建议

计划财务部 2024 年度：23 条外部义务 × 41 条内部条款 → 19 条已覆盖、3 条部分覆盖、1 条未覆盖（关联交易独立董事审议）→ 输出整改建议

SCENE 02 · 需新建

经营指标异常检测

圈定审计范围后，AI 拉取财务+运营数据，三维比对（历史趋势/预算达成/同业对比），异常点自动检索归因。

执行流程

① 异常检测 Skill（需新建）从指标库拉取全量数据并清洗
② 三维比对：自身历史 ±2σ / 预算达成 / 同业横比
③ 异常点检索关联事件（制度变更、组织调整、重大合同）
④ 输出异常清单（偏离度 + 归因 + 置信度 + 核实方向）

某子公司 2024：人均产出 ↓26%（核心团队离职，置信度 85%）· 应收账款周转天数高于同业 35 天（合同纠纷，置信度 90%）· 费用率超预算 23% 等 5 项异常

经济责任审计辅助

输入干部姓名和任期，AI 从会议纪要、合同、审批流中提取全部决策事项，逐条判合规，生成决策清单。

执行流程

① 信息提取 Skill 调取干部任职数据与审批权限
② 跨文档检索 Skill 抽取任期内全部决策文档
③ 场景推理 Skill 逐条判合规（三重一大/回避/权限）
④ 生成重大决策清单 + 合规判定表 + 简报框架

某子公司总经理任期：47 次决策会议、128 份签批合同 → 逐条检查三重一大程序与审批权限 → 标注 3 项存疑决策需补充材料

SCENE 04 · 需依赖链数据

制度修订影响审计

新制度发布后，AI 自动检出修订差异，遍历依赖链找出受影响的下游制度和流程，回溯基于旧制度做出的历史决策。

执行流程

① 文档比对 Skill 检出修订版与上一版差异
② 影响分析 Skill 遍历依赖链，找出受影响的下游
③ 逐条比对下游条款，标注冲突/断链/需更新
④ 检索基于旧制度的审批与决议，标注需重新审视项

《预算管理办法》第 7 条修订 → 4 份下游制度受影响、2 份操作手册需更新、3 笔历史审批在新制度下需重新审视

历史审计报告智能检索

开展新项目前，AI 检索所有历史审计报告，按问题类型聚类，标注整改状态，提示本次审计应重点关注的领域。

执行流程

① 跨文档检索 Skill 检索所有历史审计报告
② 按问题类型聚类统计，识别高频薄弱环节
③ 关联整改台账，标注整改状态与纸面整改嫌疑
④ 生成审计参考摘要与重点关注建议

某子公司采购业务：过去 3 年 5 份审计报告 → "采购流程合规性"出现 4 次、2 次整改未闭环 → 提示"重点关注围标串标风险"

SCENE 06 · 需系统对接

整改跟踪与闭环验证

审计组录入整改事项后，AI 自动跟踪整改进展，到期提醒，整改完成后自动比对整改报告与原始审计问题，判定闭环状态。

执行流程

① Cron 定时三级提醒（截止前 15 天 / 3 天 / 超期告警）
② 文档比对 Skill 对比整改报告 vs 原审计问题
③ 合规对齐 Skill 检查整改后是否与新制度一致
④ 判定闭环状态：已闭环 / 部分闭环 / 纸面整改 / 超期未改

"制度修订"整改项实际只改了标题未改条文 → AI 判定为"纸面整改" → 自动触发二次整改流程，推送至审计组和被审计部门

审计方案智能生成

输入被审计单位和审计类型，AI 综合历史问题、制度变更、单位特点，自动推荐审计重点和抽样策略，生成方案框架。

执行流程

① 跨文档检索 Skill 提取历史问题高发领域并聚类统计
② 信息提取 Skill 检索审计期间制度变更热点
③ 场景推理 Skill 交叉分析（历史 × 变更 × 单位特征）
④ 生成方案框架（依据 + 目标 + 范围 + 抽样 + 资料清单）

某子公司经责审计：分析 8 份历史报告 + 3 项制度变更 → 推荐重点审计三重一大决策程序、关联交易合规性、干部选任程序 → 附审计依据与抽样方法
