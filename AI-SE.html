AI 驱动软件研发机制 · 四区架构

外网 AI 创造 + AI 审查 → 安全摆渡 → 内网适配 → 自动部署 ｜ 工具链：VS Code + Kilo + OpenCode + DeepSeek / GLM

外网研发区 — 工具链

  • • VS Code + Kilo：实时代码补全与辅助（高频交互）
  • • OpenCode CLI：自主编码 Agent，批量任务执行
  • • AI 代码审查：安全漏洞、性能、规范自动检查 → 人工确认
  • • AI API 网关：统一对接 DeepSeek 和 GLM

安全传输区 — 四道检查

  • • 代码安全扫描：Semgrep / CodeQL（SAST）
  • • 依赖审查：已知 CVE、恶意包、许可证冲突
  • • 敏感信息检测：密钥、密码、内网地址
  • • 合规检查：许可证 + 出口管制 + 人工审查（高危变更）

内网适配区 — 核心原则

  • • 外网创造、内网适配 — 只做环境替换，不重新开发
  • • 适配脚本自动化：一次编写，每次传输自动执行
  • • 内网 AI 按需部署：本地轻量模型做理解和验证
  • • 内网集成测试确保适配后功能正确

内网部署区 — CI/CD 自动化

  • • GitLab CI / Jenkins：Pipeline as Code
  • • AI 审查节点：部署风险评估 + 异常检测
  • • Harbor / Nexus：内网私有制品仓库
  • • 多环境：开发 → 测试 → 预发布 → 生产
